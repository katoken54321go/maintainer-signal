# maintainer-signal

`maintainer-signal` is a small CLI that turns repository health signals into an evidence packet for open source maintainers.

It checks for the basics that make a project easier to trust, maintain, and explain: README, license, contribution guide, security policy, issue templates, CI, tests, recent commit activity, and agent guidance. The first output is a Markdown report that maintainers can use as a starting point for funding applications, stewardship reviews, or project health checkups.

## Why This Exists

Open source maintainers often need to explain the same things repeatedly:

- What does this project do?
- Who depends on it?
- Is it actively maintained?
- What would extra maintainer capacity unlock?
- How would automation or AI assistance be used responsibly?

`maintainer-signal` does not fake adoption or decide eligibility for any program. It helps maintainers collect truthful signals and spot missing repository hygiene before they publish or apply.

## Install

For local development:

```bash
git clone https://github.com/katoken54321go/maintainer-signal.git
cd maintainer-signal
npm test
npm run snapshot
```

To run directly from the repo:

```bash
node bin/maintainer-signal.js --repo . --out maintainer-signal-report.md
```

## Usage

```bash
maintainer-signal --repo /path/to/repo
maintainer-signal --repo /path/to/repo --format json
maintainer-signal --repo /path/to/repo --since-days 180 --out report.md
```

## What It Checks

- Repository basics: README, license, contributing guide, code of conduct, security policy, changelog, issue templates, and `AGENTS.md`.
- Maintenance signals: git availability, recent commits, latest commit, release tags, GitHub Actions workflows, test script, and check script.
- Application evidence draft: a short template for describing maintainer role, user impact, adoption evidence, and responsible use.

- Add a `--strict` mode that fails CI when core maintainer hygiene is missing.

- Add localization for Japanese maintainer reports.

## Responsible Use

Use this tool to make real maintenance work more visible. Do not use it to inflate claims, fabricate usage, or submit misleading applications. The best evidence is public, reproducible, and connected to real users.
